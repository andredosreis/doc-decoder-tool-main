## Como criar conta de aluno para testes (ADR-011)

> Decisao arquitectural: alunos acedem EXCLUSIVAMENTE via convite administrativo.
> Nao existe auto-registo para alunos. Ver ADR-011.

Para criar uma conta de aluno de teste, ha dois caminhos:

### Opcao A — Via painel admin (recomendado para testes manuais)

1. Fazer login em `/auth/admin-login` com conta admin
2. Ir a "Alunos" no sidebar → clicar "Convidar Aluno"
3. Preencher email ficticio: `aluno-e2e-test@appxpro.com`
4. O sistema envia email com link de configuracao de password
5. Seguir o link no email → definir password → login em `/auth/student-login`

### Opcao B — Via Supabase Dashboard (mais rapido para CI/testes automatizados)

1. Aceder ao Supabase Dashboard → Authentication → Users
2. Clicar "Add User" → preencher email + password (marcar "Auto Confirm User")
3. Na tabela `user_roles`, inserir: `{ user_id: <uuid>, role: 'user' }`
4. Na tabela `purchases`, inserir: `{ user_id: <uuid>, product_id: <uuid>, status: 'approved' }`
5. Usar essas credenciais nos testes S04–S07

### Opcao C — Via Supabase CLI (local dev)

```bash
supabase db execute --local -c "
INSERT INTO auth.users (id, email, encrypted_password, email_confirmed_at, role)
VALUES (gen_random_uuid(), 'aluno-e2e@test.com', crypt('Aluno@2026', gen_salt('bf')), now(), 'authenticated');
"
```
